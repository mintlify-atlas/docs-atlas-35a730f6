title: "Quickstart"
description: "Get from zero to an active session in under five minutes."
---

<Steps>

<Step title="Prerequisites">

Before you begin, make sure you have the following:

- **Node.js 18+** — the project uses Vite 7 and React 19, both of which require a modern Node runtime.
- **A Supabase project** — you'll need your project URL and anon key. Create one at [supabase.com](https://supabase.com) if you don't have one yet.
- **The repository cloned** locally:

```bash
git clone https://github.com/k3vosx/ProyectoBackus.git
cd ProyectoBackus
```

</Step>

<Step title="Install dependencies">

<CodeGroup>

```bash npm
npm install
```

```bash yarn
yarn install
```

```bash pnpm
pnpm install
```

</CodeGroup>

This installs React 19, Supabase JS, `@dnd-kit/core` for drag-and-drop, Tailwind CSS, and all other project dependencies.

</Step>

<Step title="Set up environment variables">

Copy the example file and fill in your values:

```bash
cp .env.example .env
```

Then open `.env` and set each variable:

| Variable | Description |
|---|---|
| `VITE_SUPABASE_URL` | Your Supabase project URL — found in **Project Settings → API**. Example: `https://xxxx.supabase.co` |
| `VITE_SUPABASE_ANON_KEY` | Your project's public anon key — found alongside the URL in **Project Settings → API** |
| `VITE_TABLE_VIAJES` | Name of the main truck trips table (default: `viajes_camiones`) |
| `VITE_TABLE_INCIDENCIAS` | Name of the incidents table (default: `incidencias`) |
| `VITE_TABLE_ROLES` | Name of the user roles configuration table (default: `configuracion_roles`) |
| `VITE_VIEW_PRIORIDAD` | Name of the priority queue view (default: `vista_unidad_prioridad`) |
| `VITE_VIEW_TURNOS` | Name of the shift dashboard view (default: `vista_dashboard_turnos`) |
| `VITE_VIEW_PROMEDIO` | Name of the net yard average view (default: `vista_promedio_patio_neto`) |

<Warning>
Never commit your `.env` file to version control. The `.env.example` file is safe to commit — it contains only placeholder values.
</Warning>

</Step>

<Step title="Start the dev server">

```bash
npm run dev
```

Vite will start a local server, typically at `http://localhost:5173`. Open that URL in your browser.

</Step>

<Step title="Log in">

The app opens directly to the **Gestión de Camiones** login screen. Enter your credentials and click **Ingresar**.

- **Correo electrónico** — your registered email address (e.g. `correo@backus.com`)
- **Contraseña** — your account password or PIN

Authentication calls `loginUsuario` in `supabaseService`, which looks up the user record in the `configuracion_roles` table and returns the user's name and role. On success, the app passes that role directly to `SimuladorMapa` via `App.tsx`:

```tsx
// src/App.tsx
if (!usuario) {
  return <ModalLogin onLogin={(_, u) => setUsuario(u)} />;
}

return (
  <SimuladorMapa
    rolInicial={usuario.rol as Rol}
    nombreUsuario={usuario.nombre ?? usuario.email}
    onLogout={() => setUsuario(null)}
  />
);
```

There are two roles:

| Role | Indicator | Capabilities |
|---|---|---|
| **Administrador** | 👑 Purple | Full access — configure simulator mode and edit traffic-light time thresholds |
| **Cliente** | 👁 Blue | Read-only monitoring view — fixed time thresholds, real-time mode only |

<Note>
If you see "Correo o contraseña incorrectos", verify that your user record exists in the `configuracion_roles` table in Supabase. The **Ingresar** button stays disabled until both fields have a value.
</Note>

</Step>

<Step title="Configure the session">

After a successful login, the **⚙ Configuración del Simulador** modal appears before the main yard view loads. What you see depends on your role.

**If you are an Administrator (👑):**

- **Modo de tiempo** — choose between two modes:
  - `⚡ Simulación (seg)` — thresholds are in **seconds**, useful for quickly testing alert cycles.
  - `🕒 Tiempo Real (min)` — thresholds are in **minutes**, for live yard operations.
- **🟡 Amarillo** — the number of seconds or minutes a truck can wait before its status turns yellow.
- **🔴 Rojo** — the threshold at which status turns red (must be greater than the yellow value).

**If you are a Client (👁):**

Mode is locked to **Tiempo Real** and thresholds are fixed at system defaults:

| State | Threshold |
|---|---|
| 🟢 Verde | ≤ 60 min |
| 🟡 Amarillo | 61 – 120 min |
| 🔴 Rojo | ≥ 121 min |

Click **🚀 Iniciar** to confirm and open the yard map.

<Tip>
For a demo or onboarding walkthrough, use **Simulación** mode with a yellow threshold of `30` seconds and a red threshold of `60` seconds — trucks will cycle through all three alert states within a couple of minutes.
</Tip>

</Step>

<Step title="Assign your first truck to a bay">

The queue panel on the left (or bottom on mobile) shows trucks waiting to be assigned. Each card displays the truck's plate, type, operation, and current wait-time alert color.

<Tabs>

<Tab title="Desktop">

Drag-and-drop is powered by `@dnd-kit/core`.

1. **Click and hold** a truck card in the queue.
2. **Drag** it over an available bay on the yard map — the bay highlights when it accepts the drop.
3. **Release** to assign the truck.

In real mode, this calls `actualizarBahiaDirecto` which updates the `viajes_camiones` table in Supabase immediately. In simulation mode, the assignment is local only and does not write to the database.

The truck card disappears from the queue and appears inside the target bay, with its alert timer running.

</Tab>

<Tab title="Mobile">

Drag-and-drop is replaced by a two-tap selection flow on screens narrower than 1024 px.

1. **Tap a truck card** in the queue panel — it becomes selected (highlighted).
2. **Tap the target bay** on the yard map to assign the selected truck to it.

The assignment follows the same real/simulation logic as desktop.

<Note>
On very compact screens (under 768 px), the yard map and queue panel are stacked vertically. Scroll down past the queue to see the full bay layout.
</Note>

</Tab>

</Tabs>

Once assigned, the truck's on-site timer starts. The bay indicator turns 🟡 yellow and then 🔴 red as the configured thresholds are crossed. To mark a truck as departed, open its card and use the exit action — in real mode this calls `marcarSalidaDirecto`, which records the departure in Supabase.

</Step>

</Steps>
