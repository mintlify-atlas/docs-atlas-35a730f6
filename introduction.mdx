## What is Backus Simulador de Camiones?

Backus Simulador de Camiones is a web application built for Backus logistics operations. It provides real-time visibility into yard activity: which trucks are waiting in the queue, which bays are occupied, how long each unit has been waiting, and how many incidents have occurred per shift.

The application connects to Supabase for live data and supports two operating modes:

- **Simulation mode** — generates fictitious trucks locally for training and testing, without touching production data.
- **Real mode** — polls the `viajes_camiones` table every 6 seconds and reflects actual yard state.

## Key capabilities

<CardGroup cols={2}>
  <Card title="Traffic-light queue alerts" icon="traffic-light">
    Each truck card turns yellow or red when wait time exceeds configurable thresholds, so operators know which unit needs attention first.
  </Card>
  <Card title="Drag-and-drop bay assignment" icon="hand-pointer">
    Drag a truck card from the queue and drop it onto any compatible bay. The system validates truck type, operation, and product before accepting the drop.
  </Card>
  <Card title="Incident tracking" icon="triangle-exclamation">
    Register up to three incidents per truck. Incident durations are automatically deducted from the net average yard time reported in the dashboard.
  </Card>
  <Card title="Shift reports" icon="chart-bar">
    View counts of completed units per shift (T1 07:00–15:00, T2 15:01–23:00, T3 23:01–06:59) and the day's average net yard time.
  </Card>
  <Card title="Simulation mode" icon="flask">
    Run a fully local simulation with randomly generated trucks to train operators or test bay configurations without any production impact.
  </Card>
  <Card title="Mobile support" icon="mobile">
    On touch devices, tap a truck to select it, then tap a bay to assign it — no drag required.
  </Card>
</CardGroup>

## Architecture overview

```
Browser (React + TypeScript + Tailwind)
    │
    ├── SimuladorMapa — main orchestrator component
    │       ├── Header — controls, role display, start/stop
    │       ├── BahiaOverlay × 10 — interactive bay slots on the yard map
    │       ├── TarjetaCamion — draggable truck cards in the queue
    │       └── PanelFlotante × 4 — draggable dashboard panels
    │
    └── supabaseService.ts — all Supabase I/O in one module
            ├── viajes_camiones — truck journey records
            ├── incidencias — incident log
            ├── usuarios — user auth table
            └── Supabase views — pre-aggregated dashboard data
```
