`src/Componentes/bahiasConfig.ts` exports the static configuration for every bay in the yard. This file is the single source of truth for bay names, truck compatibility, permitted products, visual layout, and special alerts.

## `BahiaConfig` interface

```ts
interface BahiaConfig {
  nombre: string;
  camionesPermitidos: TipoCamion[];
  tareas: { D: string[]; C: string[] };
  alerta?: string;
  posX: number;  // percentage of map width
  posY: number;  // percentage of map height
}
```

<ResponseField name="nombre" type="string" required>
  Human-readable bay label shown in the UI (e.g. `'Bahía 3'`).
</ResponseField>

<ResponseField name="camionesPermitidos" type="TipoCamion[]" required>
  Array of truck type codes that may dock at this bay. The drag-drop validator rejects trucks whose `tipoCodigo` is not in this list.
</ResponseField>

<ResponseField name="tareas" type="{ D: string[]; C: string[] }" required>
  Permitted product codes keyed by operation type. `D` = Descarga, `C` = Carga. An empty array means that operation is not performed at this bay.
</ResponseField>

<ResponseField name="alerta" type="string">
  Optional warning message displayed in the bay overlay when a truck is assigned. Only set for bays that require special coordination.
</ResponseField>

<ResponseField name="posX" type="number" required>
  Horizontal map position as a percentage of the container width. Applied as `left: {posX}%` in CSS. Calibrated against a 1061 × 580 px reference layout.
</ResponseField>

<ResponseField name="posY" type="number" required>
  Vertical map position as a percentage of the container height. Applied as `top: {posY}%` in CSS. Calibrated against a 1061 × 580 px reference layout.
</ResponseField>

## `getColorEstado()`

Returns the hex color string for a given `EstadoAlerta` value. Used by truck cards and bay overlays to render the traffic-light semaphore.

```ts
export const getColorEstado = (estado: EstadoAlerta): string => {
  if (estado === 'rojo')     return '#ef4444';
  if (estado === 'amarillo') return '#eab308';
  return '#22c55e';
};
```

| `EstadoAlerta` | Hex color   | Tailwind equivalent |
|----------------|-------------|---------------------|
| `verde`        | `#22c55e`   | `green-500`         |
| `amarillo`     | `#eab308`   | `yellow-500`        |
| `rojo`         | `#ef4444`   | `red-500`           |

## Type mapping — `parseTipo()`

The `parseTipo()` function in `supabaseService.ts` converts the raw `tipo_unidad` string from the database into a `TipoCamion` code. The mapping is case-insensitive and trims whitespace.

```ts
const TIPO_MAP: Record<string, TipoCamion> = {
  parihuelero:  'P',
  pariholero:   'P',   // common misspelling
  'pari-holero': 'P',  // hyphenated variant
  jumbo:        'J',
  'bi-tren':    'B',
  bitren:       'B',   // no-hyphen variant
  tolva:        'T',
};
```

| Raw DB value       | Mapped code | Notes                        |
|--------------------|-------------|------------------------------|
| `'parihuelero'`    | `P`         |                              |
| `'pariholero'`     | `P`         | Alternative spelling         |
| `'pari-holero'`    | `P`         | Hyphenated variant           |
| `'jumbo'`          | `J`         |                              |
| `'bi-tren'`        | `B`         |                              |
| `'bitren'`         | `B`         | No-hyphen variant            |
| `'tolva'`          | `T`         |                              |
| *(any other value)*| `O`         | Falls back to Otros          |
